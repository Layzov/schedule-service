Инструкция по запуску:
1) Установить make, если не установлен. Чтобы запустить Makefile :)
2) (В папки никакие переходить не нужно) При первом запуске написать make start (запуститься go mod download/tidy, docker-compose, применение миграций к бд, билд самого проекта и запуск сервера)
3) Когда бд запущена, таблицы готовы и тд. повторно запускать проект через make run.
4) Все остальные команды можно посмотреть через make help



Во время разработки проекта было принятно несколько решений, не указанных в API:
  1) Добавлены 500-ые и отличные от API 400-ые ошибки, такие как: internal server error, {} is empty.
     Валидация входных данных и обработка внутренней ошибки сервера позволила легче работать над проектом в дальнейшем.
      тк. не требовалось учитывать пустые ячейки в таблице при внесении данных. А 500 ошибка помогала при фиксе багов, которые возникали по ходу разработки.
    2) Добавлены ДТО-шные структуры (дополнительные), чтобы со слоя БД до хэндлеров не тянуть огромную вереницу из данных. В некоторых случаях, представленных в апи дто не хватало.
     3) Добавлен slogpretty для улучшения читабельности логов
     4) Добавлен graceful-shutdown
     5) Организованы транзакции к бд там, где это нужно.
